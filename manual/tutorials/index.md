# Tutorials

Tutorials are for learning by doing. They assume you may be new to AIVI and even new to
functional programming, so they focus on one path at a time and aim for a concrete success.

## Recommended order

| Read this | Why it comes next |
| --- | --- |
| [Start Here](/guide/getting-started) | Learn the vocabulary: values, functions, signals, pipes, and markup. |
| [Build a Small Task Tracker](/guide/your-first-app) | Put the core ideas together in one practical GTK app. |
| [Snake](/guide/building-snake) | See the same ideas scaled up into a larger, game-shaped example. |

## What tutorials are for

Use the tutorial path when you want help answering questions like:

- *How does AIVI feel in practice?*
- *How do values, types, signals, and UI fit together?*
- *What does an idiomatic AIVI app look like from top to bottom?*

If you already know the outcome you want and just need the recipe, jump to
[How-to Guides](/how-to/). If you need the exact surface details, go to
[Reference](/reference/).
